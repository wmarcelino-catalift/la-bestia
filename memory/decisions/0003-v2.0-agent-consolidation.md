# ADR-0003 — La Bestia v2.0: 12 archetype-grounded agents + /flow orchestration

**Status:** accepted
**Date:** 2026-05-03
**Deciders:** wmarcelino-catalift
**Tags:** agents | orchestration | breaking-change | governance
**Supersedes (partial):** ADR-0002 (the agent set defined there)

## Context

By v1.1 the harness had 18 agents. CTO audit revealed:

1. **Heavy overlap.** `security-auditor` + `security-reviewer` (~95% same role). `cto-strategist` + `business-analyst` + `pm` (~75% overlap on strategy/product). `uiux` + `ux-reviewer` (~70% same role).
2. **Project leak.** `content-manager` was Catalift-specific (Firestore content, recetas, WhatsApp) — leaking project context into a generic global harness.
3. **Specialty redundancy.** `mobile-reviewer` was code-reviewer with mobile specialty — better as a frontmatter / instruction.
4. **Personality inflation.** Most agents were "thin prompts" — they declared a role but lacked the deep archetype, top-tier-company framing, and frontier-knowledge that would make a senior operator choose them over typing the question to default Claude.
5. **Orchestration gap.** No commanded way to run a full feature pipeline. `/parallel-research` did fan-out, `/cto-review` did review, `/bug-hunt` did debugging — but nothing tied a Discover → Define → Develop → Deliver flow with phase gates.

Goal: consolidate to a smaller, deeper, project-agnostic set with real orchestration.

## Options considered

1. **Status quo (18 agents)** — keep, accept the overlap and project leak.
2. **Aggressive cut to 8** — fold reviewer specialties into one agent with a "specialty" param.
3. **Curated 11-12 with archetypes** — consolidate redundancies, deepen each remaining agent with 20+ years framing, project-agnostic.

## Tradeoffs

| Dimension                             | Weight | Opt 1 (18)          | Opt 2 (8)                   | Opt 3 (12)                            |
| ------------------------------------- | ------ | ------------------- | --------------------------- | ------------------------------------- |
| Capability coverage                   | 3      | full                | partial (collapsing voices) | full                                  |
| Token weight (frontmatter at startup) | 2      | 18×~50 = 900        | 8×~50 = 400                 | 12×~75 = 900                          |
| Voice diversity                       | 3      | high but blurry     | low                         | high and sharp                        |
| Project portability                   | 3      | low (catalift refs) | high                        | high                                  |
| Orchestration support                 | 2      | none                | none                        | `/flow` skill                         |
| Migration cost for operator           | 2      | none                | high (renames)              | medium (renames + agent-memory remap) |

## Decision

**Option 3: 12 archetype-grounded agents + `/flow` orchestration skill.**

### Removals (9)

- `cto-strategist` → folded into `strategist`
- `business-analyst` → folded into `strategist`
- `pm` → folded into `strategist`
- `security-auditor` → folded into `security`
- `security-reviewer` → folded into `security`
- `uiux` → folded into `designer`
- `ux-reviewer` → folded into `designer`
- `mobile-reviewer` → folded into `code-reviewer` (mobile specialty section)
- `content-manager` → deleted (was Catalift-specific; project-local now)

### Additions (3)

- `strategist` (opus) — Bezos PR-FAQ + Christensen JTBD + Eric Ries lean + Munger inversion.
- `security` (sonnet) — Schneier + Mudge + STRIDE + OWASP ASVS + NIST Zero-Trust.
- `designer` (sonnet) — Don Norman + Apple HIG + Material 3 + WCAG 2.2 + Inclusive Design.

### Refactors (9 — same names, new depth)

Each existing agent rewritten with: 20+ years archetype intro, top-tier-company framing, explicit frontier-knowledge section, anti-patterns, chain protocol, memory-write protocol.

### Additions (orchestration)

- `config/skills/flow-feature/SKILL.md` — Discover → Define → Develop → Deliver pipeline.
- `config/commands/flow.md` — `/flow "<feature>"` activator.
- `config/hooks/route-prompt.sh` — keyword routing updated for new agent set.
- `config/CLAUDE.md` — constitution v2.0 reflects 12 agents + `/flow` canonical.

## Consequences

**Positive**

- Each agent now has a unique, defensible voice grounded in real-world top-tier practice. No more "thin prompts".
- Project-agnostic: `content-manager` removed → harness installs cleanly into any project.
- Token weight at startup roughly equal (12 × deeper frontmatter ≈ 18 × thin frontmatter), but signal-to-noise far higher.
- `/flow` provides the missing piece: a real orchestration ritual with parallel fan-out and phase gates.
- `route-prompt.sh` simpler and clearer (12 categories vs 14 with overlaps).

**Negative / accepted costs**

- v1.x → v2.0 is breaking. Operators with `~/.claude/agent-memory/<old-name>/MEMORY.md` keep their notes (we don't auto-delete) but the named-agent reference no longer exists. Migration: copy old MEMORY.md content into the new agent's MEMORY.md (e.g., `cto-strategist` + `business-analyst` + `pm` → `strategist`).
- Project-specific `content-manager` workflow needs to be replicated by the operator in their project-local `.claude/agents/` (we provide a template).
- Agent-memory directories for the 9 deleted names are orphaned. install.sh keeps them on disk (operator-private data); operator deletes manually if desired.

**Neutral / to monitor**

- Token cost per session — measure via ccusage. If `/flow` Large runs blow the budget, refine the fan-out gates in flow-feature SKILL.md.
- Voice consistency — re-read the 12 agent files together periodically to ensure the personalities don't blur.

## Failure modes

1. **Operator confused which agent to call.** Mitigation: route-prompt.sh suggests; agent table in README + constitution; `/agents` command lists.
2. **Agent personality drift over PRs.** Mitigation: reviewers use the `## Anti-patterns this agent rejects` section as ground truth.
3. **Skill `/flow` over-fans-out and burns tokens.** Mitigation: triage step gates (Small/Medium/Large + Touches) → fewer dispatches; token-saver auto-activates > 60% context.
4. **Schema rejects the new agent files.** Mitigation: CI bats + schema validation before commit.

## Reversibility

- [x] **One-way door** for the deletions (`git revert` is possible but the agent-memory rebuild is the cost).
- [x] **Two-way door** for the new agents (can be removed in v2.1 if not earning their tokens).
- [x] **Two-way door** for `/flow` (skill + command can be removed cleanly).

Off-ramp: tag v1.2 as "v1.1 + flow skill only, no agent consolidation" and revert agent set. Cost: re-importing v1.x agent prompts; agent-memory unaffected.
