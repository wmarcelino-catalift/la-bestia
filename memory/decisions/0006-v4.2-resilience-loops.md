# ADR-0006 — La Bestia v4.2: resilience & learning loops

**Status:** accepted
**Date:** 2026-05-04
**Deciders:** wmarcelino-catalift
**Tags:** resilience | learning | parallelism | confidence
**Builds on:** ADR-0002 (no-vault, no-UI), ADR-0004 (context & routing), ADR-0005 (specialists)

## Context

By v4.1, La Bestia had a hard 80% overlap with the dominant Claude Code ecosystem patterns (CLAUDE.md, specialized subagents, hooks, skills, ADRs, MCP opt-in, Plan-Apply ritual). A gap analysis against current ecosystem leaders (`awesome-claude-code-toolkit`, "Claude Code Mastery", "Better Claude Code") revealed four areas where La Bestia trailed:

1. **Context loss after `/compact`** — the #1 complaint of advanced operators across Reddit / GitHub. La Bestia's `inject-context.sh` only fires on `startup`, not on `compact` or `resume`.
2. **No "lessons learned" loop** — La Bestia has ADRs (decisions) and patterns (reusable code shapes) but no third layer for incident-driven learnings. Operators end up re-debugging the same gotcha across sessions.
3. **No worktree integration** — git worktrees are the canonical solution for parallel features. La Bestia could be invoked inside a worktree but had no first-class support.
4. **No anti-hallucination signal on agent outputs** — operators must read every line at the same vigilance. Confidence tagging is a cheap soft contract that focuses attention.

These four are independent. Any one is shippable alone. Together they close all the practical gaps without compromising La Bestia's principles (no UI, no vault, no auto-merge, no autonomous deploy).

## Options considered

For each gap, the alternatives:

### Gap 1 — context loss

1. Status quo (operator manually re-pastes hot-context after `/compact`).
2. Extend `inject-context.sh` to be source-aware.
3. New dedicated hook `restore-context.sh` that fires only on `compact|resume`.
4. Persist a "session snapshot" to disk and reload (DB-style).

### Gap 2 — lessons learned

1. Status quo (post-mortems live in commit messages or nowhere).
2. Use ADRs for everything (overloads ADRs).
3. Use patterns for everything (overloads patterns).
4. New `memory/lessons/` append-only directory + skill that prompts when triggered.
5. External system (Notion, Linear) — explicitly violates ADR-0002.

### Gap 3 — worktree paralelismo

1. Status quo (operator uses worktrees manually, La Bestia is unaware).
2. Document the pattern in MCP/README.
3. New `worktree-flow` skill + `/flow-worktree` command + 3 helper bins (`worktree-add`, `worktree-remove`, `worktree-list`).
4. Mandate worktrees for all `/flow` (overhead for single-track work).

### Gap 4 — anti-hallucination

1. Status quo (operator vigilance).
2. Hard runtime gate (CI rejects outputs without confidence tag).
3. Soft contract in agent prompts: every non-trivial output ends with `confidence: <h|m|l>` + one-sentence why.
4. Confidence-weighted routing (low-confidence outputs auto-trigger reviewer agent).

## Tradeoffs

| Gap | Choice                                     | Why                                                                                                                                                                 |
| --- | ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Option 3 (dedicated `restore-context.sh`)  | Keeps `inject-context` simple. Source-discrimination is testable. Disk persistence (option 4) violates "no external state" principle.                               |
| 2   | Option 4 (`memory/lessons/` + skill)       | Append-only is the cheapest layer. ADRs and patterns stay clean. External systems out per ADR-0002.                                                                 |
| 3   | Option 3 (worktree skill + command + bins) | Helps operators who want it; doesn't force overhead on operators who don't.                                                                                         |
| 4   | Option 3 (soft contract in prompts)        | Hard gate (option 2) creates CI friction without behavior change. Confidence-weighted routing (option 4) is interesting but premature; add later if data justifies. |

## Decision

**Ship all four as v4.2.**

### Gap 1 — `restore-context.sh`

- New SessionStart hook with `source` discrimination via stdin JSON or `CLAUDE_SESSION_SOURCE` env.
- Fires on `compact` and `resume`. Silent on `startup` (let `inject-context.sh` handle it).
- Injects: full hot-context (vs inject-context's truncated 40 lines), ADR index, pattern index, lessons index (last 5), last 8 commits.
- 14 bats test cases covering: source discrimination, content injection, anti-leak (no vault refs), perf < 500ms (non-fire) / < 1s (fire).
- Performance budget: < 400ms p99.

### Gap 2 — lessons-loop

- New `memory/lessons/` directory with `README.md` (when-to-write triggers, anti-patterns) and `memory/templates/lesson-template.md` (skeleton).
- New skill `lessons-loop` with auto-triggers ("lesson", "learned", "gotcha", "post-mortem", "won't repeat", debug session > 30 min).
- 5-step ritual: detect trigger → propose to operator → generate slug `YYYY-MM-DD-<descriptor>` → fill template (Context / Symptom / Root cause / Fix / Detection) → write.
- Promotion path: incident → lesson → pattern (if reusable) → ADR (if architectural).

### Gap 3 — worktree paralelismo

- New skill `worktree-flow` documenting full lifecycle (setup, run, iterate, merge, cleanup), combination with hooks, anti-patterns, token economy (~30-40% savings on multi-feature weeks).
- New command `/flow-worktree <slug> "<feature>"` that wraps the flow.
- Three helper scripts: `bin/worktree-add.sh` (validates kebab-case, refuses duplicates), `bin/worktree-remove.sh` (uncommitted-changes guard, `--force` override), `bin/worktree-list.sh` (table view).
- Honors `WT_BASE`, `WT_BRANCH_PREFIX`, `WT_BASE_BRANCH` env overrides.

### Gap 4 — confidence tagging

- New constitution section §6.2.
- Every non-trivial agent output ends with `confidence: <high|medium|low>` + one-sentence `why`.
- Mandatory for Strategy / Quality / Specialist tiers and for `@debugger`. Optional for domain agents on recommendations only.
- **Skipped in SCAN MODE** — the ≤200 token budget already implies a fast-pass. Adding confidence inflates it.
- Soft contract enforced by agent prompts. CI does not reject untagged outputs (yet — see Future).

## Consequences

### Positive

- Closes all four ecosystem gaps in a single ship.
- No breaking changes — every new surface is opt-in by usage.
- 5-layer memory (added `lessons/`) is a strict superset of the 4-layer.
- Worktree support unlocks 30-40% token savings for multi-feature operators (cache stays warm; no need to `/clear` and reload).
- Confidence tagging adds ≤30 tokens per output for a meaningful attention-allocation signal.

### Negative

- 9 hooks now (was 8). Slightly more to maintain in tests + CI + verify.
- 6 skills now (was 4). Eager-load metadata grows from ~400 tok to ~600 tok.
- Three new bin scripts. Six total worktree-related files. Documentation surface grows.
- Confidence tagging without runtime enforcement risks under-tagging from agents that ignore it. Mitigation: code-reviewer flags untagged Strategy/Quality outputs in PR reviews of agent files themselves.

### Neutral

- Worktree skill is opt-in. Single-track operators see no change.
- Lessons skill auto-triggers on debug session > 30 min — operators who don't write long debug sessions never see it.
- Restore-context only fires on `compact|resume`. Operators who don't `/compact` see no behavior change.

## Validation plan

- **Tests**: 14-case bats for `restore-context.bats`. Existing 70+ cases continue to pass.
- **Verify**: `verify.sh` updated to count 9 hooks + 9 bin utilities + 6 skills.
- **Manual smoke**: run `/flow-worktree experiment-foo "test"`, verify worktree created, `/flow` runs inside, `worktree-remove.sh experiment-foo` cleans up.
- **Production**: 1 week of dogfood across 3 operator workflows (single-track, multi-feature, debugging-heavy). Failure to use any of the four → revert that surface in v4.3.

## Future (out of scope for v4.2)

- **Confidence-weighted routing**: low-confidence outputs auto-trigger reviewer agent. Need data first to know if this is useful or noisy.
- **Lessons → patterns auto-promotion**: when the same lesson tag appears 3+ times, suggest promoting to a pattern.
- **`restore-context` content selection**: currently injects all ADRs / patterns / 5 lessons. Could rank by recency × relevance to the prompt at hand. Premature without telemetry.
- **Worktree-aware `/ship-it`**: detect if running inside a worktree and adjust the merge instructions.
