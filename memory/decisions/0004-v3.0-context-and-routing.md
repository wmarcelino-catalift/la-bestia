# ADR-0004 — La Bestia v3.0: Intent Map routing + rich context + onboarding wizard

**Status:** accepted
**Date:** 2026-05-04
**Deciders:** wmarcelino-catalift
**Tags:** routing | context | onboarding | token-economics

## Context

By v2.1 the harness had:

- 12 archetype-grounded agents with explicit chains.
- `/flow` orchestration with phase banners.
- `route-prompt.sh` keyword matching for routing.
- Minimal `inject-context.sh`: cwd + git + hot-context.md.
- Statusline with model + last-agent + cost-today.

Three real friction points emerged in operator usage:

1. **Routing is keyword-only.** `route-prompt.sh` matches regex on the prompt and suggests an agent. But the principal Claude already does semantic routing implicitly. Result: the suggestion is sometimes off (e.g., "tengo que decidir entre Postgres y MongoDB" → matches "decisión" but the actual call should be `@architect` not `@strategist`). The principal doesn't have an explicit _intent → agent_ table to consult deterministically.

2. **Context at session start is shallow.** The principal often has to spawn extra agents or `Read package.json` / `git log` in the first turn, paying tokens for state that could be injected at startup. For a typical session this is ~5k extra tokens per onboarding turn.

3. **New-project setup is manual.** Operator has to: edit `memory/hot-context.md` from a template, decide directory structure, write the first ADR. No bootstrap command. Friction → operators skip memory setup → 3-layer rule never kicks in → no token savings ever materialize.

## Options considered

1. **Status quo (v2.1)** — keep, accept the friction.
2. **Keyword routing v2** — improve `route-prompt.sh` regex with more patterns. Doesn't address the semantic-routing gap.
3. **Intent Map in CLAUDE.md + richer SessionStart + bootstrap wizard** — work at the right layer (constitution, hook, command), all additive.
4. **Prompt caching** (Anthropic API feature) — biggest potential token savings (~50%), but requires confirming Claude Code passes `cache_control` markers from agent files. Can't validate from operator side. **Deferred to v3.1 pending verification.**

## Tradeoffs

| Dimension           | Weight | Opt 1 (status quo) | Opt 2 (regex+) | Opt 3 (this ADR) | Opt 4 (caching)                               |
| ------------------- | ------ | ------------------ | -------------- | ---------------- | --------------------------------------------- |
| Routing quality     | 3      | low                | medium         | high             | high (orthogonal)                             |
| Token savings       | 3      | 0%                 | 5%             | 20-30%           | 50%                                           |
| Implementation cost | 2      | 0h                 | 1h             | 5h               | unknown (depends on Claude Code internals)    |
| Risk of breakage    | 3      | none               | low            | low              | medium (could degrade if cache misconfigured) |
| Reversibility       | 1      | n/a                | high           | high             | high                                          |

## Decision

**Option 3.** Ship v3.0 with Intent Map + rich context + onboarding wizard. Defer prompt caching to v3.1 after validating Claude Code's cache-control behavior.

### Concrete deliverables

**1. Intent Map** (`config/CLAUDE.md` §6.1 new section).

- Table of `intent → agent / skill / command` mappings.
- Operator's intent (implement / fix / decide / review / etc.) maps to a specific dispatch.
- The principal Claude reads this table BEFORE consulting `route-prompt.sh`.
- 16 mapped intents covering 90%+ of operator prompts.

**2. Enriched SessionStart** (`config/hooks/inject-context.sh` rewritten).

- Stack auto-detection from manifest files (Node, Python, Go, Rust, Ruby, PHP, JVM, Expo, Docker, Terraform, Firebase, Supabase, Vercel).
- Git richness: branch + uncommitted + ahead-of-upstream + last commit + modified files (top 8) + recent 5 commits.
- Cost integration: `$ today` from ccusage.
- Last 24h top agents (top 3 from previous sessions).
- Project `memory/hot-context.md` injection (preserved from v2.1).

**3. Statusline enrichment** (`config/scripts/statusline.sh`).

- Per-day and per-week `$` via `ccusage daily --json`.
- Budget alert prefix: `⚠` at $20/day, `🚨` at $50/day.
- Context window % (when input JSON includes token usage).
- Hint to `/compact` when ctx ≥ 70%.

**4. Token-saver skill v2.0** (`config/skills/token-saver/SKILL.md`).

- Threshold raised from 60% → 70% (less aggressive, more useful).
- Heuristics for new statusline alerts (`⚠ $X today` → switch to haiku).

**5. `/onboard-project` command + `bin/onboard.sh` wizard.**

- Detects stack, writes `memory/hot-context.md` draft, scaffolds `memory/{decisions,patterns,templates}`, creates `.claudeignore` + project-level `CLAUDE.md`.
- Idempotent (never overwrites existing files).
- ~10 bats tests covering idempotency, stack detection, no-overwrite invariants.

## Consequences

**Positive**

- Routing decisions become deterministic + auditable. Intent Map is one read of CLAUDE.md (likely cached by Claude internally) instead of N keyword checks.
- SessionStart context gives the principal everything it needs in turn 1. Stops the "let me Read package.json first" pattern that costs 1-3k tokens per session.
- Operators new to la-bestia get bootstrapped in 30 seconds. The 3-layer memory rule actually engages.
- Statusline alerts surface cost issues before they compound (operator sees ⚠ at $20 today instead of discovering $80 the next morning).
- Token savings estimated at 20-30% per typical session vs v2.1.

**Negative / accepted costs**

- SessionStart now produces ~500 tokens of injected context (vs ~150 in v2.1). Net win because it replaces 1-2 file reads (~2-5k tokens), but it's "always on" cost.
- Intent Map adds ~30 lines to CLAUDE.md. Principal reads constitution every session, so this is also "always on". Trade is offset by routing wins.
- `/onboard-project` adds another command (now 12). Operator surface grows.

**Neutral / to monitor**

- Statusline `$` formatting may break if ccusage output format changes. Mitigation: graceful degradation (omits the field if jq parse fails).
- Stack detection is heuristic (manifest files only). Polyglot repos may show partial. Acceptable — operator can override via project `CLAUDE.md`.

## Failure modes

1. **Intent Map drifts from real-world operator prompts.** Detection: same prompt routed to wrong agent twice. Mitigation: update the table, ship as patch release.
2. **Stack detection gives false positives** (e.g., `package.json` in a Python project for npm tooling). Mitigation: operator overrides in `<repo>/CLAUDE.md`.
3. **`onboard.sh` runs in a project with custom `memory/` structure.** Mitigation: idempotency invariant — never overwrites; tests enforce.
4. **ccusage daily JSON schema changes upstream.** Mitigation: jq queries with `// empty` fallback.

## Reversibility

- [x] **All v3.0 changes are additive.** Reverting = `git revert` of one commit. No agent/skill/command is removed; only added or refined.

Off-ramp: tag v2.2 as "v2.1 + onboard wizard only, no Intent Map / no rich SessionStart" if the latter two cause regressions. Cost: ~30min revert work.
